# TaskBoard 仕様書

## 概要
Backlog風のプロジェクト・タスク管理WEBアプリケーション。

## 技術スタック
| レイヤー | 技術 |
|---------|------|
| フロントエンド | React + TypeScript + Vite + Tailwind CSS |
| バックエンド | Node.js + Express + TypeScript |
| データベース | PostgreSQL (Supabase) |
| ストレージ | Supabase Storage（アバター画像） |
| 認証 | JWT（jsonwebtoken） |
| デプロイ | Vercel（サーバーレス） |

## MVP スコープ

### 1. プロジェクト管理
- プロジェクトのCRUD（作成・一覧・更新・削除）
- プロジェクトキー（英大文字、例: `PROJ`）— 課題キーの接頭辞に使用
- プロジェクト名（必須、最大100文字）
- プロジェクト説明（任意、最大1000文字）

### 2. 課題管理
- 課題のCRUD（作成・一覧・詳細・更新・削除）
- 課題キー: `{プロジェクトキー}-{連番}` で自動採番（例: PROJ-1, PROJ-2）
- 件名（必須、最大200文字）
- 詳細（任意、テキスト、最大10000文字）

### 3. ステータス
| ID | 名称 | 色 |
|----|------|----|
| 1 | 未対応 | グレー |
| 2 | 処理中 | ブルー |
| 3 | 処理済み | オレンジ |
| 4 | 完了 | グリーン |

### 4. 優先度
| ID | 名称 | 色 |
|----|------|----|
| 1 | 高 | レッド |
| 2 | 中 | オレンジ |
| 3 | 低 | グレー |

### 5. 種別
| ID | 名称 |
|----|------|
| 1 | タスク |
| 2 | バグ |
| 3 | 要望 |
| 4 | その他 |

### 6. 担当者・登録者
- ユーザーマスタから選択
- 担当者: 任意（未割り当て可）
- 登録者: 課題作成時に自動設定（ログインユーザー）

### 6.5. アバター
- ユーザーはプロフィール画像をアップロード可能
- 対応形式: PNG, JPG, GIF（最大2MB）
- Supabase Storageの `avatars` バケットに保存
- 未設定時は名前のイニシャルをカラーサークルで表示

### 7. 期限日
- 開始日（任意）
- 期限日（任意）
- 期限日の前後関係バリデーション（開始日 ≤ 期限日）

### 8. 課題一覧・検索・フィルタ
- 一覧表示（テーブル形式）
- フィルタ条件:
  - ステータス（複数選択可、ボタントグル）
  - 優先度（複数選択可、ボタントグル）
  - 種別（単一選択、ドロップダウン）
  - 担当者（単一選択、ドロップダウン）
  - キーワード（件名の部分一致）
- ソート: 作成日（デフォルト）/ 更新日 / 優先度 / 期限日
- フィルタリセットボタン（フィルタ適用時に表示）

### 9. カンバンボード
- ステータス別の列表示（未対応 | 処理中 | 処理済み | 完了）
- 課題カードをドラッグ&ドロップでステータス変更
- カード表示内容: 課題キー、件名、優先度アイコン、担当者アバター

### 10. 完了理由
- ステータスを「完了」にした際に理由を記録する
- 完了以外のステータスでは空（NULL）

| ID | 名称 |
|----|------|
| 1 | 対応済み |
| 2 | 対応しない |
| 3 | 重複 |
| 4 | 再現しない |
| 5 | 仕様通り |

### 12. 親子課題
- 課題に親子関係を設定できる（最大2階層: 親→子）
- 親課題は同一プロジェクト内の課題のみ指定可能
- 子課題を持つ課題は他の課題の子にはなれない（孫課題は不可）
- 親課題が削除された場合、子課題の親参照はNULLに戻る
- 課題一覧ではツリー表示（親の下に子をインデント表示）
- 課題モーダルで親課題を選択可能

### 11. コメント
- 課題ごとにコメントを投稿・一覧表示
- コメント投稿者はユーザーマスタから（MVPではシードユーザー固定）
- 新しいコメントが上に表示（降順）
- コメントの削除はコメント作成者のみ（MVP時点では全員削除可）

## データモデル

### users
| カラム | 型 | 制約 |
|--------|-----|------|
| id | SERIAL | PK |
| name | TEXT | NOT NULL |
| email | TEXT | NOT NULL, UNIQUE |
| password | TEXT | NOT NULL, DEFAULT '' |
| avatar_url | TEXT | NULL可（Supabase Storageのファイル名） |
| created_at | TIMESTAMPTZ | DEFAULT NOW() |

### projects
| カラム | 型 | 制約 |
|--------|-----|------|
| id | SERIAL | PK |
| project_key | TEXT | NOT NULL, UNIQUE, 英大文字2-10文字 |
| name | TEXT | NOT NULL |
| description | TEXT | DEFAULT '' |
| created_at | TIMESTAMPTZ | DEFAULT NOW() |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() |

### issues
| カラム | 型 | 制約 |
|--------|-----|------|
| id | SERIAL | PK |
| project_id | INTEGER | FK → projects.id ON DELETE CASCADE, NOT NULL |
| issue_number | INTEGER | NOT NULL（プロジェクト内連番） |
| subject | TEXT | NOT NULL |
| description | TEXT | DEFAULT '' |
| status_id | INTEGER | NOT NULL, DEFAULT 1 |
| priority_id | INTEGER | NOT NULL, DEFAULT 2 |
| type_id | INTEGER | NOT NULL, DEFAULT 1 |
| assignee_id | INTEGER | FK → users.id, NULL可 |
| created_by | INTEGER | FK → users.id, NOT NULL |
| start_date | TEXT | NULL可 |
| due_date | TEXT | NULL可 |
| parent_issue_id | INTEGER | FK → issues.id ON DELETE SET NULL, NULL可（親課題のID） |
| resolution_id | INTEGER | NULL可（完了時のみ設定） |
| created_at | TIMESTAMPTZ | DEFAULT NOW() |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() |

**ユニーク制約**: (project_id, issue_number)

**課題キー**: `projects.project_key || '-' || issues.issue_number`（表示時に結合、DBには保存しない）

### comments
| カラム | 型 | 制約 |
|--------|-----|------|
| id | SERIAL | PK |
| issue_id | INTEGER | FK → issues.id ON DELETE CASCADE, NOT NULL |
| user_id | INTEGER | FK → users.id, NOT NULL |
| content | TEXT | NOT NULL |
| created_at | TIMESTAMPTZ | DEFAULT NOW() |

## API設計

### プロジェクト
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/projects | プロジェクト一覧 |
| POST | /api/projects | プロジェクト作成 |
| GET | /api/projects/:id | プロジェクト詳細 |
| PUT | /api/projects/:id | プロジェクト更新 |
| DELETE | /api/projects/:id | プロジェクト削除 |

### 課題
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/projects/:projectId/issues | 課題一覧（フィルタ・ソート対応） |
| POST | /api/projects/:projectId/issues | 課題作成 |
| GET | /api/projects/:projectId/issues/:issueId | 課題詳細 |
| PUT | /api/projects/:projectId/issues/:issueId | 課題更新 |
| DELETE | /api/projects/:projectId/issues/:issueId | 課題削除 |
| PATCH | /api/projects/:projectId/issues/:issueId/status | ステータス変更（カンバン用） |

### コメント
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/issues/:issueId/comments | コメント一覧（降順） |
| POST | /api/issues/:issueId/comments | コメント投稿 |
| DELETE | /api/comments/:commentId | コメント削除 |

### ユーザー
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/users | ユーザー一覧 |

### アバター
| メソッド | パス | 説明 |
|---------|------|------|
| POST | /api/avatars/upload | アバター画像アップロード（multipart, 2MB制限, PNG/JPG/GIF） |

### ヘルスチェック
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/health | DB接続確認・環境変数チェック（認証不要） |

### 認証
| メソッド | パス | 説明 |
|---------|------|------|
| POST | /api/auth/login | ログイン（JWT発行） |
| GET | /api/auth/me | ログインユーザー情報取得 |

**認証方式**: JWT（Bearer トークン、有効期限7日）。`/api/auth` と `/api/health` 以外の全APIは認証必須。トークンはフロントエンドの `localStorage` に保存し、401レスポンス時に自動ログアウト。パスワードは平文比較（MVP簡易実装）。

## 画面構成

### レイアウト
```
+------------------+------------------------------------------+
| サイドバー        | メインコンテンツ                          |
| - プロジェクト一覧 |                                          |
|                  |                                          |
+------------------+------------------------------------------+
```

### デザイン
- Backlogに倣い、緑を基調としたカラースキーム（ブランドカラー: `#42CE9F`）
- Tailwind CSSのカスタムカラー `brand` パレットで統一
- サイドバー: ダークグリーン（`brand-900`）、アクセント: `brand-400`

### 画面一覧
| 画面 | 表示方式 | 説明 |
|------|----------|------|
| ログイン | 専用ページ | メール+パスワード認証、テストアカウント表示 |
| 課題一覧 | メイン | テーブル形式（親子ツリー表示対応）、リスト/ボード切替 |
| カンバン | メイン | D&Dでステータス変更 |
| 課題詳細 | 専用ページ | Backlog風の2カラムレイアウト（左: 本文・子課題・コメント、右: 属性パネル） |
| 課題追加 | モーダル | 新規課題作成フォーム |
| プロジェクト設定 | モーダル | プロジェクトのCRUD |
